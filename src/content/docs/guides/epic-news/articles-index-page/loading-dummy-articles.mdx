## Load Articles in a `loader` function

Eventually, we will need to load the articles from our database. We will do this firstly from the `app/routes/news.$category.tsx` route file.

Open this file now in your code editor.

### Adding a `loader` function

If you remember from a [**previous lesson**](/code-monster/guides/epic-news/light-and-dark-mode/02-adding-a-loader-function#what-is-a-loader-function), a Remix `loader` function is used to execute code on the server before a page is rendered in the browser.

This is the perfect place to load data ready for a page to rendered.

### Tips

- Remix has [**a `json` helper function**](https://remix.run/docs/en/main/utils/json) that can be used to return JSON data from a loader function.
- Properties inside any JavaScript object are separated by commas, for example:

    ```tsx mark=","
    { name: 'John', age: 25 }
    ```

    Remember you need to include both the `mockArticles` array and the `category` string as properties of the object you are returning via the `json` helper function.
- In JavaScript, if an object has a property with the same name and value, you can simply omit the value.

    For example,

    ```tsx
    { category: category }
    ```

    can be written simply as

    ```tsx
    { category }
    ```
